# AGENTS.md

이 저장소에서 작업할 때 Codex(에이전트)는 아래 규칙을 따릅니다.

## 목표
- 사용자의 요청을 정확하고 실용적으로 해결합니다.
- 불필요한 변경 없이 필요한 파일만 수정합니다.

## 작업 원칙
- 변경 전 관련 파일을 먼저 읽고 맥락을 확인합니다.
- 가능한 한 작은 단위로 수정하고, 이유가 명확한 경우에만 구조를 크게 바꿉니다.
- 파일 검색은 우선 `rg`/`rg --files`를 사용합니다.
- 기존 스타일과 규칙(코드 스타일, 폴더 구조, 네이밍)을 우선 존중합니다.

## 코드 수정
- 요구사항과 직접 관련된 코드만 수정합니다.
- 복잡한 로직에는 짧고 핵심적인 주석만 추가합니다.
- 대규모 자동 생성 결과를 임의로 추가하지 않습니다.

## 검증
- 가능하면 테스트/린트/빌드를 실행해 변경 사항을 검증합니다.
- 실행하지 못한 검증이 있으면 이유를 명확히 보고합니다.

## 커뮤니케이션
- 진행 상황과 결과는 한국어로 간결하게 공유합니다.
- 최종 보고에는 다음을 포함합니다.
  - 무엇을 변경했는지
  - 왜 변경했는지
  - 검증 결과(또는 미실행 사유)

## 금지 사항
- 사용자 요청 없이 파괴적 명령(`reset --hard`, 무분별한 파일 삭제 등)을 실행하지 않습니다.
- 관련 없는 파일을 임의로 수정하지 않습니다.
